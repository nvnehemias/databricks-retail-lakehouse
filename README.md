# Databricks Retail Lakehouse Analytics Project

## Project Overview

This project demonstrates an end-to-end retail analytics workflow using Databricks, PySpark, SQL, Delta Lake, Tableau, Git, and GitHub.

The objective was to simulate a modern analytics engineering pipeline by ingesting raw ecommerce data, transforming it with Spark, storing cleaned datasets as Delta tables, and building business intelligence dashboards for stakeholder reporting.

---

## Tech Stack

- Databricks
- PySpark
- Spark SQL
- Delta Lake
- Tableau Public
- Git & GitHub

---

## Dataset

Olist Brazilian E-Commerce Dataset

---

## Project Workflow

1. Raw CSV ingestion into Databricks
2. Spark DataFrame transformations
3. Data cleaning and enrichment
4. Revenue calculations and customer segmentation
5. Delta table creation
6. SQL analytics and window functions
7. Tableau dashboard development
8. GitHub project deployment

---

## Key Analytics Performed

### Customer Segmentation
Customers were categorized into:
- High Value
- Mid Value
- Low Value

based on total revenue contribution.

### Revenue Trend Analysis
Analyzed revenue distribution and customer purchasing behavior.

### SQL Window Functions
Used ranking functions to identify top customers by revenue.

## Business Insights

- Credit card payments generated the highest revenue contribution.
- A small percentage of customers contributed disproportionately high revenue.
- Revenue distribution showed strong customer spending concentration.
- Customer segmentation can support targeted retention strategies.
